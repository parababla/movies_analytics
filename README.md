# Анализ MovieLens. Исследование пользовательских предпочтений и характеристик фильмов

## Описание задачи

В рамках данного проекта был проведён комплексный анализ набора данных MovieLens, содержащего информацию о фильмах, пользовательских оценках, тегах и ссылках на внешние ресурсы.

Цель работы: создать модуль для анализа данных MovieLens.

#### В рамках проекта реализовано:
- Разработано четыре класса для работы с файлами датасета: `Movies`, `Ratings`, `Tags`, `Links`
- Созданы тесты с использованием PyTest для проверки корректности работы методов
- Обработаны возможные исключения (несуществующие файлы, неверная структура данных)


## Ход выполнения работы

### Класс Movies (анализ фильмов)

Класс предназначен для загрузки и анализа данных из файла `movies.csv`, который содержит идентификаторы фильмов, их названия и жанры.

**Методы класса:**

| Метод | Описание |
|-------|----------|
| `__init__(path_to_the_file)` | Конструктор класса.|
| `read_file()` | Читает CSV файл, пропускает заголовок, обрабатывает каждую строку через `str_handler()`. |
| `str_handler(string)` | Парсит строку CSV. Возвращает список из трёх элементов: `[movieId (int), title (str), genres (list)]`.|
| `extract_genres(row)` | Преобразует строку жанров в список строк. |
| `extract_year()` | Извлекает год выпуска из названия фильма. Возвращает список годов. |
| `dist_by_release()` | Возвращает словарь с распределением фильмов по годам выпуска. |
| `dist_by_genres()` | Возвращает словарь с распределением фильмов по жанрам. |
| `most_genres(n)` | Возвращает словарь из `n` фильмов с наибольшим количеством жанров. Ключ — название фильма, значение — количество жанров. |

---

### Класс Tags (анализ пользовательских тегов)

Класс предназначен для загрузки и анализа данных из файла `tags.csv`, который содержит пользовательские теги, присвоенные фильмам.

**Методы класса:**

| Метод | Описание |
|-------|----------|
| `__init__(path_to_the_file)` | Конструктор класса. |
| `read_file()` | Читает CSV файл, проверяет корректность заголовка и типов данных в каждой колонке. Возвращает список строк. |
| `select_tags(lines)` | Извлекает теги из третьей колонки каждой строки. Возвращает список тегов. |
| `most_words(n)` | Возвращает словарь из `n` тегов с наибольшим количеством слов. |
| `longest(n)` | Возвращает список из `n` самых длинных тегов по количеству символов. |
| `most_words_and_longest(n)` | Возвращает пересечение между n самыми популярными тегами с наибольшим количеством слов внутри и n самыми длинными тегами по количеству символов.|
| `most_popular(n)` | Возвращает словарь из `n` самых часто встречающихся тегов. |
| `tags_with(word)` | Возвращает список уникальных тегов, содержащих заданное слово (регистронезависимо). |

### Класс Ratings (анализ пользовательских оценок)

Класс предназначен для загрузки и анализа данных из файла `ratings.csv`, который содержит оценки пользователей. Внутри класса реализованы два вложенных класса: `Movies` и `Users`.

**Методы класса Ratings:**

| Метод | Описание |
|-------|----------|
| `__init__(path_to_the_file)` | Конструктор класса. |
| `str_handler(string)` | Парсит строку CSV. |
| `read_file()` | Читает CSV файл, пропускает заголовок. |

**Вложенный класс Ratings.Movies:**

| Метод | Описание |
|-------|----------|
| `__init__(outer_instance, path_to_file)` | Конструктор. Создаёт объединённую таблицу. |
| `tables_union()` | Объединяет данные о фильмах и оценках. Создаёт словарь `movieId -> [title, genres]` и добавляет эту информацию к каждой оценке. Результат: `[userId, movieId, rating, timestamp, title, genres]`. |
| `extract_year_from_timestamps(time)` | Преобразует Unix timestamp в год. |
| `dist_by_year()` | Возвращает словарь с распределением оценок по годам. |
| `dist_by_rating()` | Возвращает словарь с распределением по значениям рейтингов. |
| `top_by_num_of_ratings(n)` | Возвращает словарь из `n` фильмов с наибольшим количеством оценок. |
| `get_average(new_list)` | Вычисляет среднее арифметическое списка. |
| `get_median(new_list)` | Вычисляет медиану списка. |
| `top_by_ratings(n, metric)` | Возвращает словарь из `n` фильмов с наибольшим средним или медианным рейтингом. |
| `get_variance(my_list)` | Вычисляет дисперсию списка. |
| `top_controversial(n)` | Возвращает словарь из `n` фильмов с наибольшей дисперсией оценок. |

**Вложенный класс Ratings.Users (наследуется от Ratings.Movies):**

| Метод | Описание |
|-------|----------|
| `__init__(movies_instance)` | Конструктор. |
| `dist_by_users()` | Возвращает словарь с распределением пользователей по количеству оставленных оценок. |
| `dist_by_metric(metric)` | Возвращает словарь с распределением пользователей по среднему или медианному рейтингу (параметр `metric='average'` или `'median'`). |
| `top_controversial(n)` | Возвращает словарь из `n` пользователей с наибольшей дисперсией их оценок. |

### Класс Links (интеграция с IMDB)

Класс предназначен для загрузки данных из файла `links.csv` (содержит `movieId`, `imdbId`, `tmdbId`) и парсинга дополнительной информации с веб-страниц IMDB.

**Методы класса:**

| Метод | Описание |
|-------|----------|
| `__init__(path_to_the_file, path_to_movie)` | Конструктор. |
| `read_file()` | Читает CSV файл, проверяет заголовок, тип данных в колонках. |
| `select_title(path_to_movie)` | Создаёт экземпляр класса `Movies` и строит словарь `movieId : title`. |
| `select_movieId(lines)` | Извлекает первую колонку (`movieId`) из каждой строки. |
| `select_imdbId(lines)` | Извлекает вторую колонку (`imdbId`) из каждой строки. |
| `select_tmdbId(lines)` | Извлекает третью колонку (`tmdbId`) из каждой строки. |
| `get_imdb(list_of_movies, list_of_fields)` | Возвращает список списков `[movieId, field1, field2, ...]`. |
| `imdb_parsing(imdbId, field)` | Выполняет HTTP-запрос к странице `https://www.imdb.com/title/tt{imdbId}/`. Парсит HTML с помощью BeautifulSoup. Поддерживаемые поля: `Director`, `Budget`, `Gross worldwide`, `Runtime`, `Title`. Для числовых значений использует регулярное выражение `r'[\d,]+'`. |
| `top_directors(n, list_of_movies)` | Возвращает словарь из `n` режиссёров с наибольшим количеством фильмов в переданном списке. |
| `most_expensive(n, list_of_movies)` | Возвращает словарь из `n` фильмов с наибольшим бюджетом. |
| `most_profitable(n, list_of_movies)` | Возвращает словарь из `n` фильмов с наибольшей прибылью (мировые сборы минус бюджет). |
| `longest(n, list_of_movies)` | Возвращает словарь из `n` фильмов с наибольшей продолжительностью (в минутах). |
| `top_cost_per_minute(n, list_of_movies)` | Возвращает словарь из `n` фильмов с наибольшим отношением бюджета к продолжительности. |

### Класс Tests

Для проверки корректности работы всех методов создан класс `Tests` с тестами, покрывающими:

**Тесты для класса Movies:**
- `test_read_file` — проверка типов данных загруженной таблицы
- `test_extract_genres` — корректность извлечения жанров
- `test_str_handler` — правильность парсинга CSV строк
- `test_extract_year` — извлечение годов из названий
- `test_dict_by_release` — тип результата и сортировка
- `test_dict_by_genres` — тип результата и сортировка
- `test_most_genres` — корректность при различных `n` (включая `n=0`)

**Тесты для класса Tags:**
- `test_select_tags` — проверка типа извлечённых тегов
- `test_most_words` — тип результата и сортировка
- `test_longest` — тип результата и сортировка по длине
- `test_most_words_and_longest` — проверка пересечения множеств
- `test_most_popular` — тип результата и сортировка
- `test_tags_with` — проверка фильтрации по слову

**Тесты для класса Links:**
- `test_select_methods` — проверка типов идентификаторов
- `test_get_imdb` — структура возвращаемых данных (с пропуском при недоступности IMDB)
- `test_top_directors`, `test_most_expensive`, `test_most_profitable`, `test_links_longest`, `test_top_cost_per_minute` — проверка типов и сортировки

**Тесты для класса Ratings:**
- `test_ratings_str_handler` — проверка типов при парсинге
- `test_ratings_read_file` — структура загруженных данных
- `test_movies_dist_by_year` — проверка сортировки по возрастанию лет
- `test_movies_dist_by_rating` — проверка сортировки по возрастанию рейтингов
- `test_movies_top_by_num_of_ratings` — корректность при различных `n`
- `test_movies_top_by_ratings_average` и `test_movies_top_by_ratings_median` — проверка метрик
- `test_movies_top_controversial` — проверка расчёта дисперсии
- `test_users_dist_by_users` — распределение пользователей
- `test_users_dist_by_metric_average` и `test_users_dist_by_metric_median` — метрики пользователей
- `test_users_top_controversial` — дисперсия оценок пользователей
- `test_edge_cases` — граничные случаи (`n=0`, запрос большего количества)
- `test_invalid_metric` — обработка некорректного параметра `metric`
